Getting Start

1. What is MongoDB/Mongoose ?

MongoDB is a schema-less NoSQL document database. It means you can store JSON documents in it, and the structure of these documents can be different as it is not enforced like SQL databases. This is one of the advantages of using NoSQL as it speeds up application development and reduces the complexity of deployments.

Mongoose is an Object Data Modeling (ODM) library for MongoDB and Node.js. Mongoose is an abstraction over the native MongoDB driver, it is a modeling tool help us to work with MongoDB environment asynchronously.

Mongoose manages relationships between data, provides schema validation, and is used to translate between objects in code and the representation of those objects in MongoDB. Here's some definitions or Terminologies that we need to understand before we start learning MongoDB/Mongoose

  • Collections: in Mongo are equivalent to tables in relational databases. They can hold multiple JSON documents.
  • Document: are equivalent to records or rows of data in SQL. While a SQL row can reference data in other tables, Mongo documents usually combine that in a document.
  • Collections: in Mongo are equivalent to tables in relational databases. They can hold multiple JSON documents.
  • Fields: or attributes are similar to columns in a SQL table.
  • Schema: As we know MongoDB is schema-less, SQL defines a schema via the table definition. A Mongoose ‘schema’ is a document data structure (or shape of the document) that is enforced via the application layer.
  • Models: are higher-order constructors that take a schema and create an instance of a document equivalent to records in a relational database.

3. Import and Connect to MongoDB

First we include mongoose in our project, using require('mongoose') module and create an instance from it. We will use the mongoose instance to create a database connection to our "DBSample" database on our locally running instance of MongoDB. 

  const mongoose = require('mongoose'); // reference to mongoose

  // Connect to a MongoDB server running on 'localhost:27017' and use the 'DBSample' database.
  mongoose.connect('mongodb://localhost:27017/DBSample')
  .then(() => {
    console.log('Database connection successful')
  })
  .catch(err => {
    console.error('Database connection error')
  })

To connect to MongoDB from Node.js using Mongoose package, call connect() function, on the variable referencing to mongoose, with MongoDB Database URI passed as argument to the function. 

  const mongoose = require("mongoose")

  async function db_connect() {
   try {
    // Connect to a MongoDB server running on 'localhost:27017' and use the 'DBSample' database.
    await mongoose.connect("mongodb://localhost:27017/DBSample")
    console.log("Database is connected successfully..")
   } catch (error) {
    console.log("Error:", err)
   }
  }

4. Define Schema

Schema define the structure of the document with properties through an object where the key name corresponds to the property name in the collection. In your schema you can provide default values, validators to validate the received value. With Mongoose, everything is derived from a Schema, so lets create our first schema. 

  const mongoose = require("mongoose")

  // connecting to MongoDB
  mongoose.connect("mongodb://localhost:27017/DBSample")
  ....

  const personSchema = new mongoose.Schema({
    username: String, //  Define username field of type string
    email: { //  Define email field of type string and it's a mandatory field and it must be unique
      type: String,
      required: true,
      unique: true
    },
    age: Number, // Define age field of type Number
    isMarried: Boolean // Define isMarried field of type Boolean
  });
