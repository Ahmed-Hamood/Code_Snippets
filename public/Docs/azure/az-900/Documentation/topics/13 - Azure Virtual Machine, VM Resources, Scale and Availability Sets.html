Azure Virtual Machine, VM Resources, Scale & Availability Sets

A - What is Virtual Machine

Virtual machine is a software emulation of physical computer that azure offers . VMs provide infrastructure as a service (IaaS) in a form of a virtualized server with resources which is comprised of:

  • CPU Processor
  • RAM Memory
  • NIC - Network Interface card
  • HD or SSD storage

Virtualization is an emulation of physical machines that create virtual machines with different Operating System and virtual hardware. Each virtual machine has it's own configuration and OS. We can say its a Total separation of OS environments.

  • File systems
  • Services
  • Ports
  • Middleware
  • Configuration

With VM you take a subset or smaller part of CPU, RAM or HD/SSD to create a virtual machine that will run a separate operating system from the actual server.

VMs like a physical computer they host an operating system and you can install, run and customize all of the software running on your VM.

Azure VMs gives you flexibility of virtualization without having to buy and maintain the physical hardware that runs the VM. As you know VMs provide IaaS offering, which mean you still need to maintain the virtual machine by performing tasks, such as configure, updating/patching and maintaining/installing the software that runs on it.

On VM you can run application like web app and web services, databases, desktop application, jumpboxes, gateways, etc.

VMs are great choice when you need here some use cases:

  • Total control over the operating system (OS).
  • The ability to run custom software for developing and testing.
  • To use custom hosting configurations.
  • Extend your data-center with hybrid cloud
