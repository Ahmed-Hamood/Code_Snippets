1. Application Hosting Options

In order to host an applications then we probably need VM or container for the hosting solution. As we know VMs is the best hosting method and it gives you a full control over the hosting environment and allow to configure it exactly how you want.

Container is a compelling hosting solution option for isolating apps and individually manage each app separately with it own environment

What if i tell you that you can host your apps without the need of managing infrastructure like creating VMs and deploying Containers, Azure provides app services that will do all the hosting for us. You build your app with all packages and codes and send it to azure app service, and azure will host it for you.

- Azure App Service

Azure app service is a simple an HTTP-based service for hosting web applications (different runtimes and programming languages), it enables you to build and host web apps, background jobs, mobile back-ends, and RESTful APIs in the programming language of your choice without managing infrastructure.

Azure app service is PaaS offering, the main purpose of this service is building web applications. We just ask developers to prepare a simple deployment package/code and send it to app service.

App Service supports Windows and Linux. It enables automated deployments from GitHub, Azure DevOps, or any Git repo to support a continuous deployment model. Azure App Service lets you focus on building and maintaining your app, and Azure focuses on keeping the environment up and running.

App Service handles most of the infrastructure decisions you deal with in hosting web-accessible apps :-

  • Deployment and management are integrated into the platform.
  • Endpoints can be secured.
  • Sites can be scaled quickly to handle high traffic loads.
  • The built-in load balancing and traffic manager provide high availability.

App service offers automatic scaling and high availability features. You can scale up to 20 or 100 nodes (Vms underneath) depend on the pricing tier.

The pricing based on app service plan, a set of compute resources is created for that plan in that region. Each App Service plan defines: Regions, Number of VM instances, size of VMs and pricing tier. The pricing tier of an App Service plan determines what App Service features you get and how much you pay for the plan.

Three Pricing Tier are currently available :-

  • Shared Compute Dev/Test (Free and Shared) – The app is run on the same VM as other apps; scaling is not an option. This plan is good for development and testing environment. Ideal for less demanding workloads. Focused on providing shared infrastructure. Additional features include custom domains / SSL and manual scale.
  • Dedicated compute (Basic, Standard, Premium, Premium v2) Production – Apps are run on a dedicated VMs; apps in the same App Service plan shared the same compute resources. Ideal for more demanding workloads. Additional features include staging slots, daily backups, and a traffic manager.
  • Isolated - dedicated VMs run in dedicated vNETs. Ideal for workloads that require advanced networking and fine-grained scaling.

- With App Service, you can host most common web app styles, such as :

Web Apps - build and deploy web apps faster at scale. App Service includes full support for hosting web apps by using ASP.NET, ASP.NET Core, Java, Ruby, Node.js, PHP, or Python. You can choose either Windows or Linux as the host operating system.

- Available run-times, programming languages and containers :

API Apps – easily build and consume APIs. You can build REST-based web APIs by using your choice of language and framework.

Web App for containers - deploy and run containerized web apps.

WebJobs - WebJobs are often used to run background tasks as part of your application logic. You can use the WebJobs feature to run a program (.exe, Java, PHP, Python, or Node.js) or script (.cmd, .bat, PowerShell, or Bash) in the same context as a web app, API app, or mobile app. They can be scheduled or run by a trigger.

Mobile apps - Use the Mobile Apps feature of App Service to quickly build a back end for iOS and Android apps. With just a few actions in the Azure portal, you can:

  • Store mobile app data in a cloud-based SQL database.
  • Authenticate customers against common social providers, such as MSA, Google, Twitter, and Facebook.
  • Send push notifications.
  • Execute custom back-end logic in C# or Node.js.
