Azure Virtual Machine - VM Resources Types and Sizes

1. What is Virtual Machine

Virtual machine is a software emulation (Virtualization) of computer system that azure offers . VMs provide infrastructure as a service (IaaS) in a form of a virtualized server with resources which is comprised of:

  • CPU Processor
  • RAM Memory
  • NIC - Network Interface card
  • HD or SSD storage

Virtualization is an emulation of physical machines that create virtual machines with different Operating System and virtual hardware. Each virtual machine has it's own configuration and OS. We can say its a Total separation of OS environments. (read more)

  • File systems
  • Services
  • Ports
  • Middleware
  • Configuration

With VM you take a subset or smaller hardware part of CPU, RAM or HD/SSD to create a virtual machine that will run a separate operating system from the actual server.

A piece of software called a hypervisor, or virtual machine manager, lets you run different operating systems on different virtual machines at the same time. This makes it possible to run Linux VMs, for example, on a Windows OS, or to run an earlier version of Windows on more current Windows OS.

VMs like a physical computer they host an operating system and you can install, run and customize all of the software running on your VM. VM are independent of each other, and they are extremely portable. You can move a VM on a hypervisor to another hypervisor on a completely different machine almost instantaneously.

Azure VMs gives you flexibility of virtualization without having to buy and maintain the physical hardware that runs the VM. As you know VMs provide IaaS offering, which mean you still need to maintain the virtual machine by performing tasks, such as configure, updating/patching and maintaining/installing the software that runs on it.

On VM you can run application like web app and web services, databases, desktop application, jumpboxes, gateways, etc.

Some common examples or use cases for virtual machines include:

  • Total control over the operating system (OS): You can create or run your own custom software/environment for developing and testing and use custom hosting configurations.
  • During testing and development: We can easily create and delete VMs, with VMs we can create different OS and application configuration to code and test an application.
  • Running a traditional infrastructure app: we can run certain applications in the public cloud and run traditional app in your own premises, you can use public cloud on demand, shutting down VMs when you don't need them or quickly starting them up to meet a sudden increase in demand means you pay only for the resources you use.
  • When extending your data-center to the cloud : An organization can extend the capabilities of its own on-premises network by creating a virtual network in Azure and adding VMs to that virtual network.
  • Lift and shift: Moving from physical data-center to cloud. You can take image of the server & run within a VM with little to no changes.
  • During disaster recovery (Backing up your existing OS): As with running certain types of applications in the cloud and extending an on-premises network to the cloud, you can get significant cost savings by using an IaaS-based approach to disaster recovery. If a primary data-center fails, you can create VMs running on Azure to run your critical applications and then shut them down when the primary data-center becomes operational again.

virtual machines provide many benefits, such as :-

  • Cost savings - running multiple virtual environments from one piece of infrastructure means that you can drastically reduce your physical infrastructure footprint. This boosts your bottom line-decreasing the need to maintain nearly as many servers and saving on maintenance costs and electricity.
  • Agility and speed - Spinning up a VM is relatively easy and quick and is much simpler than provisioning an entire new environment for your developers. Virtualization makes the process of running dev-test scenarios a lot quicker.
  • Lowered downtime - VMs are so portable and easy to move from one hypervisor to another on a different machine - this means that they are a great solution for backup, in the event the host goes down unexpectedly.
  • Scalability - VMs allow you to more easily scale your apps by adding more physical or virtual servers to distribute the workload across multiple VMs. As a result you can increase the availability and performance of your apps.
  • Security benefits - Because virtual machines run in multiple operating systems, using a guest operating system on a VM allows you to run apps of questionable security and protects your host operating system. VMs also allow for better security forensics, and are often used to safely study computer viruses, isolating the viruses to avoid risking their host computer.

- VM image

A virtual machine image is a single file or template which contain a virtual disk that has a boot-able operating system installed on it with other software like development tools or web hosting environment.

With azure you can create or use/choose an already created image (pre-configured) from the microsoft market to rapidly provision VMs in minutes.

Azure let you choose/use VM either from Microsoft virtual machine images from the Microsoft marketplace, or use your own VM system that include OS and other software, with your own configuration, installed services and run-times (create by developers) for your company. And then you put this custom image in your custom or marketplace storage.

Azure virtual machine grabs those images either choose a custom or marketplace image to start creating a new virtual machine and exposing them to your users in just couple of minutes a virtual machine will be provisioned and ready to use.

2. VM Resources - Types, Sizes and Managed Disk Storage

Azure has limited number of VMs on per subscription, The current limit on a per subscription basis is 20 virtual machines per region. Limits can be raised by filing a support ticket requesting an increase.

- Types and Sizes in Azure

Azure VMs are available in different sizes and types that able to serve you in different cases. When you provision a VM, you will have a chance to pick the resources that are associated with that VM. resource types and sizes options like:

  • Size (like number of processor cores, and amount of RAM)
  • Storage disks (like hard disk drives, solid state drives, etc.)
  • Networking (like virtual network, public IP address, and port configuration)

The size of the virtual machine that you use is determined by the workload that you want to run. The size that you choose then determines factors such as processing power, memory, storage capacity, and network bandwidth. Azure offers a wide variety of sizes to support many types of uses as you see below.

Azure charges an hourly price based on the virtual machine's size and operating system. For partial hours, Azure charges only for the minutes used. Storage is priced and charged separately.

- Managed Disk Storage

Azure managed disks are block-level storage volumes that are managed by azure and used with azure virtual machines. Managed disks are like a physical disk in an on-premises server but they are virtualized; with managed disks, all you have to do is specify the disk size, the disk type like performance tier (Standard or Premium) and azure will provision the disk.

Managed Disks handles Azure Storage account creation and management in the background for you, and ensures that you do not have to worry about the scalability limits of the storage account. If you're creating new virtual machines, use the Azure CLI or the Azure portal to create virtual machines with Managed OS and data disks.

If you have virtual machines with un-managed disks, you can convert your virtual machines to be backed with Managed Disks. Also you can also manage your custom images in one storage account per Azure region, and use them to create hundreds of virtual machines in the same subscription.
