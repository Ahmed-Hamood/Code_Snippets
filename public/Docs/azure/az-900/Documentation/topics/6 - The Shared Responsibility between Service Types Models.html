- The Shared Responsibility between Service Types Models

1 - Shared responsibility model Intro

In your company you have on premises data-center, the company is responsible for maintaining the physical space, ensuring security, and maintaining or replacing the servers if anything happens.

The IT department in your company is responsible for maintaining all the infrastructure and software needed to keep the data-center up and running. They're also likely to be responsible for keeping all systems patched and on the correct version.

With the shared responsibility model, these responsibilities get shared between the cloud provider and the consumer. Using cloud computing, you don't need to build a traditional corporate data-center and make large upfront investments in hardware and spend a lot of time provisioning the hardware with exactly the right type and size of computing resources you need to power and run your services.

The Cloud provider is responsible for Physical security, power, cooling, and network connectivity. The consumer isn't collocated with the data-center, so it wouldn't make sense for the consumer to have any of those responsibilities.

The consumer is responsible for the data and information stored in the cloud. And also, responsible access security to give access to those who need it. On-premises data-center, you're responsible for everything. But with cloud provider those responsibilities shift

- Cloud service types consumer and cloud provider responsibilities :-

  • IaaS - infrastructure as a service: places the most responsibilities on the consumer, the cloud provider only responsible for the basics of physical security, power, and connectivity.
  • PaaS - Platform as a service: being a middle ground between IaaS and SaaS in the middle it evenly distributes responsibility between the cloud provider and the consumer.
  • SaaS - Software as a service: most of the responsibility with the cloud provider, and the consumer only responsible for data information and account access. It places most of the responsibility with the cloud provider.

- You as a consumer/admin user always be responsible for :-

  • The information and data stored in the cloud
  • Devices that are allowed to connect to your cloud (cell phones, computers, and so on)
  • The accounts and identities of the people, services, and devices within your organization

- The cloud provider is always responsible for :-

  • The physical data-center
  • The physical network
  • The physical hosts

- Your service model will determine responsibility for things like :-

  • Operating systems
  • Network controls
  • Applications
  • Identity and directory infrastructure

A - IaaS - Shared responsibility model

IaaS places the largest share of responsibility with you as consumer. The cloud provider is responsible for maintaining the physical infrastructure and its access to the internet. Also you're responsible for installation and configuration, patching and updates, and security.

Ownership in IaaS, the Cloud provider manage infrastructure like networking, hardware and virtualization. You as a user/admin you manage platform and software. IaaS means is having a lot of virtual machines, virtual networks and managed disks.

- You as a consumer you're responsible for :-

  • Platform - operating system, middleware and runtime.
  • Software - data and application.

The use case is migration of workloads from the on-premises environment, also you can use for testing and development. And having more capacity by extending your on-premises environment by using storage with backups and recovery in case of any disasters.

B - PaaS - Shared responsibility model

PaaS splits the responsibility between you and the cloud provider. The cloud provider is responsible for maintaining the physical infrastructure and platform and its access to the internet, just like in IaaS.

  • Infrastructure - networking, hardware and virtualization
  • Platform - operating system, middleware and runtime

In the PaaS model, the cloud provider will also maintain the operating systems, databases, and development tools. Think of PaaS like using a domain joined machine: IT maintains the device with regular updates, patches, and refreshes.

Depending on the configuration, you or the cloud provider may be responsible for networking settings and connectivity within your cloud environment, network and application security, and the directory infrastructure.

As a user/admin you manage software data and applications, you will only need to manage your own application, it is used whenever you're developing cloud applications like :

  • building Web applications
  • Artificial intelligence (AI).
  • Machine learning (ML)

C - SaaS - Shared responsibility model

SaaS is the model that places the most responsibility with the cloud provider and the least responsibility with the user. You're responsible for the data that you put into the system, the devices that you allow to connect to the system, and the users that have access.

- Cloud provider manages infrastructure, platform and software and you manage nothing :-

  • Infrastructure - networking, hardware and virtualization
  • Platform - operating system middleware and runtime.
  • Software - data and applications

Nearly everything else falls to the cloud provider. The cloud provider is responsible for physical security of the data-centers, power, network connectivity, and application development and patching.

D - On-Premises

Here cloud provider manage nothing, you manage everything in your organization. You manage Infrastructure, platform and software.

  • Infrastructure - networking, hardware and virtualization
  • Platform - Operating system, middleware and runtime
  • Software - data and applications
