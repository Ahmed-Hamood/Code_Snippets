5 - Cloud Service Types Models - IaaS, PaaS, SaaS

As a service means which party will manage the particular layer and all the layers below.

  • Infrastructure layer consists hardware the infrastructure and virtualization required to host the platform
  • Platform layer means all the supporting software and the operating system required to host the application
  • Software layer consists the application (application code and set) & the application data

A - Infrastructure as a Service - IaaS (for Admins)

IaaS provides you the maximum amount of control for your cloud resources. It provides the highest level of flexibility and management control over the infrastructure. No need to worry about IT Staff, data center and hardware.

IaaS Aims to give you the most control over the provided hardware that runs your application, and instant computing infrastructure, provisioned and managed over the internet. You rent hardware instead of buying.

Cloud provider ensure that a service is available up and running, and configure and functioning correctly is a shared responsibility that should provide to their customers.

- IaaS is literally renting IT infrastructure like :-

  • Servers
  • Virtual Machines (VMs)
  • Storage space
  • Networks
  • Operating Systems

The only responsible for cloud provider is maintaining the hardware, network connectivity (to the internet), and physical security. And you ensures services are configured correctly up to date and available to your customers/clients.

Your responsibility as Admin, you responsible for everything else: operating system installation, configuration, and maintenance; network configuration; database and storage configuration;

- Common IaaS use cases :-

  • Migrating workloads: Managed similar to on-prem infrastructure & provides easy migration path to IaaS infrastructure.
  • Test and development: Teams can quickly set-up & establish configurations for development and test environments that you need to rapidly replicate with fast & economical scaling.fast & economical scaling. You can stand up or shut down the different environments rapidly with an IaaS structure, while maintaining complete control.
  • Storage, backup and recovery: Organizations avoid the capital outlay and complexity of storage management.
    • Useful for managing unpredictable demand and steadily growing storage needs.
    • can also simplify the planning and management of backup and recovery systems.
