- Cloud Service Types Models - IaaS, PaaS, SaaS

- As a service means which party will manage the particular layer and all the layers below. :-

  • Software layer consists the application (application code and set) & the application data
  • Platform layer means all the supporting software and the OS required to host the application
  • Infrastructure layer consists hardware the infrastructure and virtualization required to host the platform

- The following each layer with it service (These categories are layers on top of each other) :-

  • Application - Software as a Service - SaaS
  • Data - Software as a Service - SaaS
  • Runtime - Platform as a Service - PaaS
  • Middleware - Platform as a Service - PaaS
  • Operating System -Platform as a Service - PaaS
  • Virtualization - Infrastructure as a Service - IaaS
  • Servers - Infrastructure as a Service - IaaS
  • Networking - Infrastructure as a Service - IaaS
  • Storage - Infrastructure as a Service - IaaS

A - Infrastructure as a Service - IaaS (for Admins)

IaaS provides you the maximum amount of control for your cloud resources. It provides the highest level of flexibility and management control over the infrastructure. No need to worry about IT Staff, data center and hardware.

IaaS Aims to give you the most control over the provided hardware that runs your application, and instant computing infrastructure, provisioned and managed over the internet. You rent hardware instead of buying.

An Example of Infrastructure as a Service like Microsoft Azure, AWS, Oracle Cloud and more.

Cloud provider ensure that a service is available up and running, and configure and functioning correctly is a shared responsibility that should provide to their customers.

- IaaS is literally renting IT infrastructure like :-

  • Servers
  • Virtual Machines (VMs)
  • Storage space
  • Networks
  • Operating Systems

The only responsible for cloud provider is maintaining the hardware, network connectivity (to the internet), and physical security. And you ensures services are configured correctly up to date and available to your customers/clients.

Your responsibility as Admin, you responsible for everything else: operating system installation, configuration, and maintenance; network configuration; database and storage configuration;

- Common IaaS use cases :-

  • Migrating workloads: Managed similar to on-prem infrastructure & provides easy migration path to IaaS infrastructure.
  • Test and development: Teams can quickly set-up & establish configurations for development and test environments that you need to rapidly replicate with fast & economical scaling.fast & economical scaling. You can stand up or shut down the different environments rapidly with an IaaS structure, while maintaining complete control.
  • Storage, backup and recovery: Organizations avoid the capital outlay and complexity of storage management.
    • Useful for managing unpredictable demand and steadily growing storage needs.
    • can also simplify the planning and management of backup and recovery systems.

B - Platform as a Service - PaaS (for Developers)

In PaaS, we focus on the development and management of your applications. Azure provide an environment for building, testing, and deploying software applications. It aims to help creating an application quickly without managing the underlying infrastructure.

- An Examples of PaaS - Platform as a Service is :-

  • Beanstalk AWS
  • Heroku
  • Google App Engine

- building cloud applications thru PaaS solutions like :-

  • SQL - relational databases
  • App service - web hosting
  • Logic Apps - enterprise integrations
  • Function App - server-less solution

With PaaS we don't worry about, provisioning, configuring or understanding the hardware and licensing or patching for Operating system and database. This is really for people that are building apps, only they can access it over a secure internet connection

For Example, for a web app / Azure SQL database you don't need to install an operating system, web server, or even system updates.

- The cloud provider maintains :-

  • The physical infrastructure and security
  • The connection to the internet
  • And also maintain the Servers, operating systems, middleware , Virtualization , Runtime , Networking, Storage and development tools
  • The business intelligence services that make up a cloud solution.

PaaS is mostly used by developer; quickly create web or mobile apps, without taking care of underlying infrastructure of servers, storage network, DBs, etc. You only purchase resources on a pay-as-you-go basis and accessed them over a secure Internet connection to start working on your applications.

- Common PaaS use cases :-

  • Development framework :-
    • Lets developers create applications using built-in software components.
    • Cloud features such as scalability, high-availability, and multi-tenant capability are included
    • Reducing the amount of coding that developers must do.
  • Analytics or business intelligence :-
    • Tools provided as a service with PaaS allow organizations to analyze and mine their data.
    • organizations can find insights and patterns, and predict outcomes to improve business decisions such as forecasting ,product design, and investment returns.

C - Software as a Service - SaaS (for Customers)

SaaS is a product that is run and managed by service provider. You don't need to worry about how the service is maintained or how the underlying infrastructure is managed, it just works and remain available for you to use it.

SaaS is the least flexible but also the easiest to get up and running, and it requires the least amount of knowledge

SaaS is a method for delivering software applications over the internet, on demand and typically on subscription basis (monthly/yearly). With SaaS, you're essentially renting or using a fully developed application.

- Common examples of a SaaS implementation of connectivity software :-

  • Email and messaging applications.
  • Business productivity applications.
  • Finance software and expense tracking.

- Also, here some IaaS Applications that you use as a customer :-

  • Gmail, outlook
  • Office 365
  • Sale-force
  • Dynamics CRM Online.
  • Skype

SaaS is centrally hosted and managed for the end customer/user. Usually customers get licensed applications through a monthly or annual subscription. All customer will have the same version of the application, IaaS is usually based on an architecture.

- Management responsibilities

  • IaaS: user is responsible for managing the operating systems, data, and applications.
  • PaaS: user is responsible for the applications and data they run and store.
  • SaaS: user just uses the software.
