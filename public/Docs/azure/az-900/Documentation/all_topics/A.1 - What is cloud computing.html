A.1 - What is Cloud Computing ?

1. Define Cloud computing

Cloud computing is a model for delivering computing services over the internet. Simply it's like using a network of remote servers hosted on the internet to store, manage and process data rather than local server or personal computer.

Cloud computing lets you choose the power and features you need to run your software. All Servers/PCs are in a cloud provider's data-center instead of physically with you. Cloud only lets you pay fot only the services you use

- Computing services which include common IT infrastructure resources and software such as: -

  • Servers with Virtual Machines and containers
  • Storage (HD, SSD)
  • Databases
  • Networking
  • Software/App
  • Analytics and intelligence
  • Also expand the traditional IT offerings to include things like Internet of Things (IoT), machine learning (ML), and artificial intelligence (AI).

Cloud computing really represents renting resources like CPU, RAM and storage from someone else data-center like cloud provider such as Azure. You only pay for what you use Pay-as-you-go pricing model.

The cloud provider takes care of maintaining the underlying infrastructure for you. Someone else gets to manage the infrastructure for you like maintenance, backup and make sure your OS is up-to-date and also make sure that everything is up and running 24hrs a day.

Cloud computing offers flexible resources and economies of scale, this means if you need to increase your IT infrastructure resources rapidly, you don't have to wait to build a new data-center.

Cloud is not constrained by physical infrastructures the same way that traditional data center is. You can treat cloud resources like you would your resources in your own data center. When you're done using them, you give them back.

  • Storage: is a volume of data you can store, with unlimited data storage that you need/request overtime. We store structured or unstructured data like files and databases. Cloud delivers a thousand of popular DB technologies. Also azure provides tools that allow you to migrate your data/app to the cloud
  • Networking: After gathering your components, we need to connect them to each other securely which our cloud provide a secure networking functionality. You also can connect azure to your company network thru private connection VPN or ExpressRoute.
  • Analytics: cloud provide a Service for analytics capabilities to review your performance with virtualization and telemetry data of your services.

We know that Cloud computing is really bunch of servers that our application and IT services runs on. Servers are comprised of CPU Processor, RAM Memory and storage HDD or SSD. An Example of Applications and IT services like:

  • Email Servers
  • Web Server (web application)
  • Database
  • FTP Servers

- Cloud provider provide on-demand access to :-

  • Access to unlimited resources and services like compute power, storage and networking components. Also cloud offers things like Internet of Things (IoT), machine learning (ML), and artificial intelligence (AI).
  • Cloud provides speech recognition and other cognitive services that help make your applications stand-out from the crowd
  • Cloud provides analytics services that enable you to track your software with telemetry data coming back from your devices.
  • Cloud provides tools for migrating your apps and data to their virtual machines.

3. What is Microsoft Azure/Azure Services?

Microsoft Azure is a cloud computing platform or cloud provider service that delivers a set of services to help you build solutions to meet your business goals.

  • Azure has a simple web service for hosting and managing your business software solution in virtualized computers fully running and presence in the cloud.
  • Azure provides a cloud-based services like remote storage, CPU compute power, database hosting and centralized account management .
  • Azure offers new capabilities like AI (Artificial Intelligence) and IoT (Internet of Things) services.

4. Benefits of Cloud Computing

With cloud you don't need to make up large investment in hardware and spend time provisioning and configure the hardware and software. Also provision exactly the right type and size of computing resources you need to power and run your services. With cloud

With cloud you don't need to do that, you can have 10 or 100 servers in minutes/instantly and pay for what your use.

The company or on-premise data-center is responsible for maintaining the physical space, ensuring security, and maintaining or replacing the servers if anything happens.

The IT department (Cloud provider) is responsible for maintaining all the infrastructure and software needed to keep the data-center up and running, also they are likely to be responsible for keeping all systems patched and on the correct version. (here some of the benefits of cloud computing )

A - Cost-Effective

You pay for what you use, no up-front-cost. It provides pay-as-you-go - PAYG or consumption-based pricing model. Cloud has a thousands of customers sharing the cost of resources, that why you are getting low cost.

  • No upfront infrastructure costs
  • No need to purchase and manage costly infrastructure/hardware that you may not use to its fullest
  • The ability to pay for additional resources only when they are needed (elasticity)
  • The ability to stop paying for resources that are no longer needed, you only pay for what you use

B - Global

With azure cloud you can go online-global in minutes, you can lunch workloads anywhere in the world, just choose a region. You can deploy your application in multiple regions around the world with just simple steps.

Deploying you application in multiple regions will provide lower latency and better experience for your customers at minimal cost.

  • Fully redundant data-centers located in various regions all over the globe.
  • Enables local presence close to your customers to give them the best response time
  • Replicate your services into multiple regions for redundancy redundancy and locality
  • Select a specific region to ensure you meet data-residency and compliance laws for your customers.

C - Secure

The provider takes care of physical and digital security, it enforce strict security policies at both physical and digital layers, with latest technologies, controls, and expert technical skills can provide better security than most organizations can otherwise achieve.

Cloud services can be secure by default, you can configure accessibility down to granular level

A - Physical security :-

  • Who can access the building, who can operate the server racks, and so on
  • Walls, cameras, gates, security personnel, employees have access only to those resources that they've been authorized to manage.

B - Digital security :-

  • Who can connect to your systems and data over the network.
  • For Example, only authorized users to be able to log into virtual machines or storage systems running in the cloud
  • Have tools to mitigate security threats that you can use.

D - Reliable/Durable/fault tolerance/Disaster Recovery

Cloud provider offers data backup, disaster recovery, and Global data replication services, and fault tolerance. Azure cloud infrastructure is built in order to accommodate disasters; If one component fails, a backup component will seamlessly take ownership, assuring services availability.

Data in your azure storage account is replicated to ensure durability and high availability.

  • Data backup if one component fails, a backup component takes its place, this is referred to as fault tolerance and it ensures that your customers aren't impacted when a disaster occurs.
  • Data replication services and Redundancy is often built into cloud services architecture.

Reliability is the ability of an item to perform its function under specified conditions over a given period. usually expressed in terms of mean time between failures (MTBF), which is the average length of time that a device will operate without failure.

Durability is the ability of an object to withstand physical damage from normal use over a long period. This may include wear and tear, corrosion, impact, etc. In some cases, durability is also referred to as "endurance".

Fault tolerance is the ability to remain up and running during component and service failures, without interrupt your services. Most services have built-in fault tolerance feature which means if you use these services, you will not notice those localized failures affecting your system

Fault tolerance might scale to become a disaster caused by natural (flood, earthquakes or under storm power gird failures) or human induced like damaging or causing system error.

Disaster recovery is the ability to recover from an event that has taken down the service. With disaster recovery we create two copies of the same application/data into separate azure regions.

Setting up replications between two regions with identical copies of your application, along with those replicated applications you need to set up a simple DNS routing , so that your users are automatically redirected to the working version of your application till the local application version recover.

E - Scalable

You can increase/decrease resources and services base on the demand or workload at any given time. Azure cloud is highly scalable and can accommodate any business growth; manual or automatic based on the workload for example specific triggers such as CPU utilization.

Azure cloud can adapt to your business by scaling either vertically (scale up/down) or horizontally (scale in/out) scaling.

Horizontal scaling (scaling in/out): Add/Remove servers that function together as one unit, this means you can add resources/servers of the same size and type. For example, add another 3, 10 or 100 virtual machines of the same size in order to sustain my business.

Vertical scaling (scaling up/down): Increase/Decrease the power of an existing server/virtual machine resources. For example, add more CPUs, or add more memory within the same machine.
