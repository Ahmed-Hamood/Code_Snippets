2. Branches Rules

We have some rules that you need to know while creating branches, lets start

A - Branch Opens downward

First branches are opened downwards, which mean when you click on power rail and create a new branch, the new branch it will be opened/created downward

- branches always open downward, it will never open upwards, so it always comes downward

B - Branch closed upward

Another rules, our branch closes upward, you only can close branch with upper branch only. You can attach branch with the branch below but it won't considered as closed, as you see below.

If you try to close from top to bottom, our branch automatically be closed upwards, and it will move the output upward. You can't create a path that comes from branch 1 down to branch 2 to output as you see below.

4. Rungs in Ladder Logic

Our Ladder is mapped in one or more networks. A network contains power rail on the left where one or more rungs originated. Rungs consist of branches, crossing's, LED elements.

We can say lines with branches are the rang. the rang can be a series of element connections, or arrangement of branches with parallel connection. Our rang is closed by a coil or a box which the result of logic operation will be written.

Rungs and networks are executed from top to bottom and from left to right. This means the first instruction in the first rung of the first network is processed first. All instructions of this rung are then processed.

The next network is proceeded only after all rungs have been run.
