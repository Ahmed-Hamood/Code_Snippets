PLC memory areas structure and function

The standard PLC system has three main parts input modules, CPU-Controller and output modules, you PLC system can work with or without a physical input/output. The CPU and memory is utilize with 5 volts.

  • Input Modules: Collect data from sensors, potentiometer or push buttons and convert them into digital/analog numbers
  • Controller/CPU: Controller updates the code loaded in the CPU with the data received from the input module, then execute the code and generates a new results
  • Output Modules: Our output module pick-up the updated results and converts them into different forms of electric signals to control devices such relays and actuators

As we know our PLC/CPU is powered by 24VDC, either is provided by internal or external power supply. For internal power supply you need 120/220V AC power source to power your PLC, this means you PLC has an internal power supply which our 120/220V AC will be converted into 24V DC.

If your PLC need a 24VDC then this means that you need to provide an external power supply that gives you 24VDC.

It's better to invest in a reliable power supply either buying an external module PS that your PLC vendor manufacture, or buy a PLC with internal power supply if the PLC suits your requirement based on the consumption or PLC system design.

- CPU Memory structure - Memory Areas

In the PLC, there are various type of CPU-Memory, you must understand how cpu memory in your PLC is organized and how to use it in tour program.

To download our program into the PLC we use a spacing user memory like SD-Card or flash storage to load your program into the CPU memory.

Memory is the component that stores information, programs, and data in a PLC. The process of putting new information into a memory location is called writing. The process of retrieving information from a memory location is called reading.

1 - Program memory content

Your program usually contains some program code blocks and data tables, our PLC store them into spacing user memory and keep the code there when the power is off.

Our program executable code will execute when the CPU is on the RUN mode. In RUN mode our PLC operating system will transfer a copy of executable parts of the program into the RAM.

2 - RAM

Ram stands for random access memory, its used to allocate memory for variables and initialize them. Also it stores executable codes which refer to machine language and set of native instructions in order to execute them.

Keep in mind the moment you power cycling or restarting your PLC/Controller, all your RAM content will be erased. On the STOP mode all variables and initializations will be reset.

PLC comes with portable RAM which means you can't change the size of your RAM in your PLC. Make sure that pick PLC with the right CPU Power and Memory Size that fits your application and requirements.

A RAM location can be read or written. This means the information stored in a RAM location can be retrieved and/or altered. Ladder logic programs are stored in RAM.

When a new ladder logic program is loaded into a PLC's memory, the old program that was stored in the same locations is over-written and essentially erased.

3 - CPU - Memory System

CPU has a dedicated memory space for inputs and outputs tables as well as timers and counters. All I/O signals are communicated to I/O cards by the memory system.

Also in the memory system our CPU record the status logs and system information such as.

  • Data and Time
  • System variable: A Variable is a container that holds information; there are many different types, and it depends on the PLC in use. The main Variable types (also known as Data Types) are:
    • BOOL: Digital Data (True / False)
    • BYTE: Numerical Data / Bitwise Data (0 - 255)
    • INT: Numerical Data (-32768 - 32767)
    • UINT: Numerical Data (0 - 65535)
    • SINT: Numerical Data (-128 - 127)
    • USINT: Numerical Data (0 - 255)
    • DINT: Numerical Data (-2147483648 - 2147483647)
    • WORD: Numerical Data / Bitwise Data (0 - 65535)
    • DWORD: Numerical Data / Bitwise Data (0 - 4294967295)
    • REAL: Numerical Data (-3.402823e+38 - 3.402823e+38)
    • ARRAY: Array of Any Data type (Declared as "ARRAY [0..10] OF DataType)
  • Most PLCs support the above, and some PLCs will support a selection of the below also:
    • LWORD: Numerical Data / Bitwise Data (0 - 18446744073709551615)
    • UDINT: Numerical Data (0 - 4294967295)
    • LINT: Numerical Data ( -9,223,372,036,854,775,808 - 9,223,372,036,854,775,807)
    • ULINT: Numerical Data (0 - 18446744073709551615)
    • VARIANT: Object (Anything)
    • NULL: Object (Nothing)

4 - Retentive Memory

Some CPU memory may offer a certain part of its memory in a retentive format, the content will be stored in a retentive memory which will not be erased or lost by CPU restart or power cycle

Retentive Memory is a good choice to save calibration data and recipe parameter, production counter and also keeping track of machine working hours.

Every time you download your application program into PLC, the content of retentive memory will be erased.

5 - ROM

A ROM location can be read, but not written. ROM is used to store programs and data that should not be altered. For example, the PLC's operating programs are stored in ROM.

- Memory Areas in S7-1200

1. Input Memory - Process image input

Here's a special memory area for physical inputs, this memory area is dedicated for inputs which will read the status of inputs and it will store its status in the memory.

The CPU copies the status of the physical inputs to I-Memory at the beginning of the scan cycle.

2. Output Memory - Process image output

Here's a special memory area for physical outputs, this memory store the values of the outputs in the memory.

The CPU copies the status of Q-Memory to the physical outputs at the beginning of the scan cycle.

3. Bit Memory

Its a special memory with two parts a general and specialized bit memories. The general memory is used for latching or making timer and also making bit memories.

The specialized bit memory has some functions like internal clocks which will be ON/OFF interval by itself and you can use it in your program logic.

The user program reads and writes the data stored in M-Memory. Any code block can access the M-Memory. You can configure addresses within M-Memory to retain the values of the data after a power cycle.

4. Temp Memory - Local memory

this memory store the inputs and outputs results in a stack, which is a group of memory allocated for making the logic for processing the information.

After the logic execution finish the stack is cleared and it can be used for other process of information.

Whenever a code block is called, the CPU allocated the temporary or local memory to be used during execution of the block. When the execution of the code block finishes, the CPU reallocates the local memory for the execution of other code block.

5. Data Block - DB

Suppose my inputs is connected to a sensor and the sensor is used to read the number of block, boxed or bottles, we use data-block to store this information.

With Data block we can use it to display information in HMI touch screen, like checking how many bottles are coming, this information is stored in data-block.

Use the DB memory for storing various types of data, including intermediate status of an operation or other control information parameters for FBs and data structures required for many instructions such as timer and counters.

You can specify a data block to be either read/write or read only. You can access data block memory in bits, bytes, words or double words.
