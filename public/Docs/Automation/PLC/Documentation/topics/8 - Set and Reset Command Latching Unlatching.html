Set and Reset Command - Latching/Unlatching

1. SET Instruction

The SET instruction is similar to the output symbol but with S letter inside. When ON signal passes into the SET instruction, the SET is activated, then the data value at the OUT address is set to 1.

As you see the moment you provide signal to SET instruction our output that you assign to it will be ON or set to 1. If you turn off the signal to our SET instruction while its ON, then the output will remain the same as ON or 1.

When SET is not activated, or no signal is provided to it, then OUT is no changed. If the OUT is ON then it remains ON

You can turn off Q0.0 output by resetting your Output address, if you cut the signal to it, it won't effect the output status and our output will still remain ON. To turn off our output we can use the RESET instruction signal.

2. RESET Instruction

The RESET instruction is similar to output symbol but with R letter. Here if RESET is activated or ON, the the data value at the OUT address is set to 0. When R is not activated, out is not changed.

Our Reset instruction works along with SET instruction, when you turn ON your Q0.0 output using the SET signal, to turn Q0.0 OFF then you activate the RESET button to turn OFF Q0.0.

With SET/RESET instruction you can use it multiple time no like the the output coil you only can use it once.

3. Example - SET and RESET Instruction

We have an example that contain NO - Normally open contact assigned with I0.0 - A letter assign to it, this I0.0 contact is connected to the SET operation of X letter, Our SET is associated with Q0.0 output.

In the brunch below, we have another NO - Normally open contact with I0.1 - B letter assign to it, this this I0.1 contact is connected to the RESET operation of X letter, Our RESET is associated with Q0.0 output also.

  • If A is False | 0 and B is False | 0, then our Q0.0 output will remain as the last state which OFF | 0.
  • If A is False | 0 and B is True | 1 (set the reset), then our Q0.0 output will be OFF | 0.
  • If A is True | 1 (set the SET) and B is False | 0 , then our Q0.0 output will be ON | 1.
  • If A is False | 0 and B is False | 0, then our Q0.0 output will remain last state which is ON | 1.
  • If A is True | 1 (set the SET) and B is True | 1 (set the SET), then our Q0.0 output will be OFF | 0.

4. Example - Two SET and RESET with different Contacts

Here we have two outputs Q0.0 and Q0.1, both are attached to I0.0 NO contact that will latch both outputs of SET.

And For Unlatching both outputs we have a separate branch with both outputs attached to I0.1 - NO, this input will be attached to both output RESET.

  • If I0.0 is False | 0, then Q0.0 and Q0.1 will remain the same as last state which is OFF
  • If I0.0 is True | 1, then both outputs Q0.0 and Q0.1 will be ON
  • If I0.0 is False | 0, then Q0.0 and Q0.1 will remain the same as last state which is ON
  • If I0.1 is True | 1, then Q0.0 and Q0.1 will reset and will be OFF
  • If both I0.0 and I0.1 are True | 1, then both outputs Q0.0 and Q0.1 will RESET to be OFF
